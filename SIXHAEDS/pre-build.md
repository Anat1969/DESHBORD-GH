# קומפוזיציה: לפני בנייה

## מתי להשתמש
- פרויקט חדש
- אפליקציה חדשה
- פיצ'ר משמעותי
- שינוי כיוון

## סדר הסוכנים

```
┌─────────────────────────────────────────────────┐
│  1. אדום (חזון)                                 │
│     → מגדיר מטרה, משתמש, סיפור                  │
│     → פלט: vision_output                        │
│                                                  │
│  2. ירוק (חדשנות)                               │
│     → קלט: vision_output                        │
│     → מייצר 3 חלופות: safe / balanced / bold     │
│     → פלט: innovation_output                    │
│                                                  │
│  3. לבן (מחקר)                                  │
│     → קלט: innovation_output                    │
│     → מאמת הנחות, אוסף נתונים לכל חלופה        │
│     → פלט: research_output                      │
│                                                  │
│  4. שחור (ביקורת)                                │
│     → קלט: innovation_output + research_output  │
│     → מעריך סיכונים לכל חלופה                    │
│     → פלט: quality_output                       │
│                                                  │
│  ════════════════════════════════════════════     │
│  נקודת החלטה: בחירת כיוון                        │
│  ════════════════════════════════════════════     │
│                                                  │
│  5. כחול (תהליך)                                 │
│     → קלט: כיוון נבחר + quality_output          │
│     → בונה ארכיטקטורה + workflow + לוח זמנים      │
│     → פלט: process_output                       │
│                                                  │
│  6. צהוב (ערך)                                   │
│     → קלט: process_output                       │
│     → מעריך ROI + מגדיר KPIs                     │
│     → פלט: value_output                         │
└─────────────────────────────────────────────────┘
```

## פלט סופי של הקומפוזיציה

```yaml
pre_build_output:
  vision: "[מ-אדום]"
  chosen_direction: "[מ-ירוק, אחרי סינון לבן+שחור]"
  validated_data: "[מ-לבן]"
  mitigated_risks: "[מ-שחור]"
  architecture: "[מ-כחול]"
  expected_roi: "[מ-צהוב]"
  go_no_go: "GO | NO-GO | PIVOT"
  ready_for: "during-build composition"
```

## כלל מנחה
**אם שחור מוצא סיכון critical — חזרה לירוק לפני שממשיכים.**
**אם לבן מפריך הנחה מרכזית — חזרה לאדום לעדכון חזון.**
