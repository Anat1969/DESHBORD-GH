# קומפוזיציה: לפני בנייה

## מתי להשתמש
- פרויקט חדש
- אפליקציה חדשה
- פיצ'ר משמעותי
- שינוי כיוון

## סדר הסוכנים

```
┌─────────────────────────────────────────────────┐
│  1. אדום (חזון)                                 │
│     → מגדיר מטרה, משתמש, סיפור                  │
│     → פלט: vision_output                        │
│                                                  │
│  2. ירוק (חדשנות)                               │
│     → קלט: vision_output                        │
│     → מייצר 3 חלופות: safe / balanced / bold     │
│     → פלט: innovation_output                    │
│                                                  │
│  3. לבן (מחקר)                                  │
│     → קלט: innovation_output                    │
│     → מאמת הנחות, אוסף נתונים לכל חלופה        │
│     → פלט: research_output                      │
│                                                  │
│  4. שחור (ביקורת)                                │
│     → קלט: innovation_output + research_output  │
│     → מעריך סיכונים לכל חלופה                    │
│     → פלט: quality_output                       │
│                                                  │
│  ════════════════════════════════════════════     │
│  נקודת החלטה: בחירת כיוון                        │
│  ════════════════════════════════════════════     │
│                                                  │
│  5. כחול (תהליך)                                 │
│     → קלט: כיוון נבחר + quality_output          │
│     → בונה ארכיטקטורה + workflow + לוח זמנים      │
│     → פלט: process_output                       │
│                                                  │
│  6. צהוב (ערך)                                   │
│     → קלט: process_output                       │
│     → מעריך ROI + מגדיר KPIs                     │
│     → פלט: value_output                         │
└─────────────────────────────────────────────────┘
```

## פלט סופי של הקומפוזיציה

```yaml
pre_build_output:
  vision: "[מ-אדום]"
  chosen_direction: "[מ-ירוק, אחרי סינון לבן+שחור]"
  validated_data: "[מ-לבן]"
  mitigated_risks: "[מ-שחור]"
  architecture: "[מ-כחול]"
  expected_roi: "[מ-צהוב]"
  go_no_go: "GO | NO-GO | PIVOT"
  ready_for: "during-build composition"
```

## כלל מנחה
**אם שחור מוצא סיכון critical — חזרה לירוק לפני שממשיכים.**
**אם לבן מפריך הנחה מרכזית — חזרה לאדום לעדכון חזון.**

# קומפוזיציה: בזמן בנייה

## מתי להשתמש
- בדיקה יומית / שבועית
- סיום שלב (milestone)
- גילוי בעיה
- בקשת שינוי

## סדר הסוכנים

```
┌─────────────────────────────────────────────────┐
│  1. כחול (תהליך)                                 │
│     → סטטוס: מה הושלם? מה בעיכוב? מה הבא?       │
│     → פלט: status_report                        │
│                                                  │
│  2. לבן (מחקר)                                  │
│     → מדידה: KPIs מול יעד, מטריקות ביצוע        │
│     → פלט: metrics_report                       │
│                                                  │
│  3. שחור (ביקורת)                                │
│     → בדיקה: בעיות חדשות? סיכונים שהתממשו?       │
│     → פלט: issues_report                        │
│                                                  │
│  4. ירוק (חדשנות)                               │
│     → התאמות: פתרונות לבעיות, שיפורים מהירים     │
│     → פלט: adjustments                          │
│                                                  │
│  ════════════════════════════════════════════     │
│  רק אם יש סטייה משמעותית מהתוכנית:              │
│  ════════════════════════════════════════════     │
│                                                  │
│  5. אדום (חזון)                                  │
│     → בדיקה: האם הכיוון עדיין נכון?              │
│                                                  │
│  6. צהוב (ערך)                                   │
│     → בדיקה: האם ה-ROI עדיין הגיוני?             │
└─────────────────────────────────────────────────┘
```

## פלט סופי

```yaml
during_build_output:
  status: "on_track | delayed | blocked | ahead"
  metrics_vs_target: "[מ-לבן]"
  new_issues: "[מ-שחור]"
  adjustments_made: "[מ-ירוק]"
  direction_change: "none | minor | major"
  next_checkpoint: "מתי הבדיקה הבאה"
```

# קומפוזיציה: אחרי מסירה

## סדר הסוכנים

```
┌─────────────────────────────────────────────────┐
│  1. צהוב (ערך)                                   │
│     → מה הערך שנוצר? ROI בפועל מול צפי?          │
│     → פלט: actual_roi                           │
│                                                  │
│  2. לבן (מחקר)                                  │
│     → נתוני שימוש אמיתיים, פידבק, מטריקות       │
│     → פלט: post_launch_data                     │
│                                                  │
│  3. שחור (ביקורת)                                │
│     → מה לא עבד? מה נשבר? מה הפתיע?             │
│     → פלט: lessons_negative                     │
│                                                  │
│  4. כחול (תהליך)                                 │
│     → תיעוד: decision log, runbook, retrospective│
│     → פלט: documentation                        │
│                                                  │
│  5. אדום (חזון)                                  │
│     → מה הלאה? עדכון חזון לגרסה הבאה            │
│     → פלט: next_vision                          │
│                                                  │
│  6. ירוק (חדשנות)                               │
│     → שיפורים, הזדמנויות חדשות, תוכנית שדרוג     │
│     → פלט: improvement_roadmap                  │
└─────────────────────────────────────────────────┘
```

```yaml
post_build_output:
  actual_roi: "[מ-צהוב]"

  next_version_vision: "[מ-אדום]"
  improvement_roadmap: "[מ-ירוק]"
  template_for_reuse: "מה אפשר לתבנת לפרויקטים הבאים"
```

# קומפוזיציה: מצב חירום

## סדר הסוכנים — מהיר וממוקד

```
┌─────────────────────────────────────────────────┐
│  1. שחור (ביקורת) — 5 דקות                       │
│     → מה בדיוק נשבר? מה החומרה? מה הרדיוס?      │
│     → פלט: incident_report                      │
│                                                  │
│  2. לבן (מחקר) — 5 דקות                         │
│     → עובדות: מתי התחיל? כמה מושפעים? logs?       │
│     → פלט: fact_sheet                           │
│                                                  │
│  3. ירוק (חדשנות) — 10 דקות                     │
│     → 3 פתרונות: מהיר/זמני, מאוזן, מלא           │
│     → פלט: solution_options                     │
│                                                  │
│  4. כחול (תהליך) — 5 דקות                        │
│     → תוכנית פעולה: מי עושה מה, עד מתי           │
│     → פלט: action_plan                          │
│                                                  │
│  ════════════════════════════════════════════     │
│  ביצוע                                           │
│  ════════════════════════════════════════════     │
│                                                  │
│  5. צהוב (ערך) — אחרי הפתרון                     │
│     → מה למדנו? מה שווה לשנות?                   │
│                                                  │
│  6. אדום (חזון) — אחרי הפתרון                    │
│     → האם צריך לשנות כיוון?                      │
└─────────────────────────────────────────────────┘
```

```yaml
crisis_output:
```
