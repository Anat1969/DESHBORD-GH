# קומפוזיציה: בזמן בנייה

## מתי להשתמש
- בדיקה יומית / שבועית
- סיום שלב (milestone)
- גילוי בעיה
- בקשת שינוי

## סדר הסוכנים

```
┌─────────────────────────────────────────────────┐
│  1. כחול (תהליך)                                 │
│     → סטטוס: מה הושלם? מה בעיכוב? מה הבא?       │
│     → פלט: status_report                        │
│                                                  │
│  2. לבן (מחקר)                                  │
│     → מדידה: KPIs מול יעד, מטריקות ביצוע        │
│     → פלט: metrics_report                       │
│                                                  │
│  3. שחור (ביקורת)                                │
│     → בדיקה: בעיות חדשות? סיכונים שהתממשו?       │
│     → פלט: issues_report                        │
│                                                  │
│  4. ירוק (חדשנות)                               │
│     → התאמות: פתרונות לבעיות, שיפורים מהירים     │
│     → פלט: adjustments                          │
│                                                  │
│  ════════════════════════════════════════════     │
│  רק אם יש סטייה משמעותית מהתוכנית:              │
│  ════════════════════════════════════════════     │
│                                                  │
│  5. אדום (חזון)                                  │
│     → בדיקה: האם הכיוון עדיין נכון?              │
│                                                  │
│  6. צהוב (ערך)                                   │
│     → בדיקה: האם ה-ROI עדיין הגיוני?             │
└─────────────────────────────────────────────────┘
```

## פלט סופי

```yaml
during_build_output:
  status: "on_track | delayed | blocked | ahead"
  metrics_vs_target: "[מ-לבן]"
  new_issues: "[מ-שחור]"
  adjustments_made: "[מ-ירוק]"
  direction_change: "none | minor | major"
  next_checkpoint: "מתי הבדיקה הבאה"
```

## כלל מנחה
**4 סוכנים ברוטינה. 6 רק כשמשהו השתנה.**
**מהירות > שלמות בבדיקות יומיות.**
